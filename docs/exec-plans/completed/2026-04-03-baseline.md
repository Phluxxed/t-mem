# Baseline Metrics Implementation

**Date**: 2026-04-03
**Goal**: `fm baseline` command that captures a pre-injection "before" snapshot for later comparison.

## Context

Tip injection has been running since ~2026-03-29. We have 2,545 pre-injection sessions in `~/.claude/projects/` and 84 post-injection sessions. We need to snapshot baseline metrics now before more post-injection sessions accumulate, so we can measure delta later.

## Metrics to Compute

From `Action` and `Turn` models (no new parsing needed):

| Metric | How |
|--------|-----|
| `error_rate` | `failed_actions / total_actions` |
| `recovery_rate` | error→success sequences on same tool / total error sequences |
| `avg_retries_after_error` | retries before success or abandonment |
| `avg_actions_per_turn` | `total_actions / total_turns` |
| `avg_turns_per_session` | `total_turns / total_sessions` |
| `session_length_p50/p90` | percentile distribution |
| `repeated_op_rate` | turns with consecutive same-tool calls / total turns |

## Command Design

```
fm baseline [--before DATE] [--sample N] [--output PATH]
```

- `--before`: cutoff date (default `2026-03-29`, when injection started)
- `--sample`: max sessions to process (default 500, `--sample 0` = all)
- `--output`: where to save snapshot (default `~/.future_memory/baseline.json`)

Sample randomly from pre-cutoff files so the snapshot is reproducible and doesn't take forever (2545 files × parse overhead = significant).

## Implementation

1. New file `src/fm/baseline.py` — metric computation logic

3. Test in `tests/test_baseline.py`

## What NOT to do

- Don't re-implement parsing — use existing `parse_session()`
- Don't store raw session data — aggregate only
- Don't compute NLP/semantic metrics — structural only (tool calls, success flags)
- No "after" comparison logic yet — just capture the snapshot
